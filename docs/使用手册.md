## 一、概述与设计思想

`giants-web` 的核心目标是**统一 WEB 层的输入输出与异常处理**，围绕三个关键点展开：

1. **统一响应结构**：无论 Spring MVC 还是 Struts2，接口最终都以统一结构返回，前端只需根据错误编码 `code` 做分支处理。
2. **异常即响应**：业务代码只管抛出语义化的异常（`GiantsException` 体系），由框架负责把异常翻译成错误码 + 国际化提示信息。
3. **API 解耦**：通过 Filter/ThreadLocal、参数解析器等手段，让业务代码尽量少地直接触碰 Servlet / Session / Cookie API。

三个模块的依赖关系：

```
giants-web-springmvc ──> giants-web, giants-common
giants-web-struts2   ──> giants-common
giants-web           ──> (仅 servlet-api / commons-lang / slf4j)
```

> 说明：`giants-web-springmvc` 与 `giants-web-struts2` 都依赖 `giants-common`（提供 `GiantsConstants`、`GiantsException` 异常体系、`Page` 分页对象等）。

## 二、giants-web 模块

WEB 公共工具类封装，不绑定任何 MVC 框架，可单独使用。

### 2.1 WebFilter：请求上下文

`com.giants.web.filter.WebFilter` 借助 `ThreadLocal` 保存当前线程的 `HttpServletRequest`，
使得在 Service 层等任意位置都能拿到当前请求，而不必层层传递 request 参数。

**配置（web.xml）**

```xml
<filter>
    <filter-name>webFilter</filter-name>
    <filter-class>com.giants.web.filter.WebFilter</filter-class>
</filter>
<filter-mapping>
    <filter-name>webFilter</filter-name>
    <url-pattern>/*</url-pattern>
</filter-mapping>
```

`WebFilter` 继承自 `AbstractFilter`，请求进入时把 request 存入 `ThreadLocal`，请求结束后（`finally`）清空，
避免线程复用导致的内存泄漏与脏数据。

### 2.2 WebUtils：请求上下文工具类

`com.giants.web.utils.WebUtils` 是对 `HttpServletRequest` 的简化调用封装。**必须先配置 `WebFilter`** 才能取到上下文。

| 方法 | 说明 |
| --- | --- |
| `getRequest()` | 获取当前线程的 `HttpServletRequest` |
| `getSessionAttribute(String name)` | 获取 Session 属性 |
| `setSessionAttribute(String name, Object value)` | 设置 Session 属性 |
| `getHeader(String name)` | 获取请求头 |
| `getIpAddress()` | 获取客户端真实 IP（自动穿透常见代理头） |
| `getIpAddress(HttpServletRequest request)` | 从指定 request 获取客户端真实 IP |

`getIpAddress()` 依次尝试 `X-Forwarded-For`、`Proxy-Client-IP`、`WL-Proxy-Client-IP`、`HTTP_CLIENT_IP`、
`HTTP_X_FORWARDED_FOR`，最后回退到 `request.getRemoteAddr()`；对多级代理的 `X-Forwarded-For`（逗号分隔）会取第一个有效 IP。

**示例**

```java
// 在任意被 WebFilter 覆盖的请求链路中
String ip = com.giants.web.utils.WebUtils.getIpAddress();
String ua = com.giants.web.utils.WebUtils.getHeader("User-Agent");
com.giants.web.utils.WebUtils.setSessionAttribute("loginUser", user);
```

### 2.3 AbstractFilter：Filter 基类

`com.giants.web.filter.AbstractFilter` 是所有 Filter 的基类，提供：

- **防重入**：同一个 Filter 在一次请求中被重复调用（如 forward 到 JSP）时不重复执行业务逻辑。
- **参数查找**：`findInitParameter(name, default)` 先查 Filter 的 `init-param`，再查全局 `context-param`，最后用默认值。
- **异常吞噬开关**：`eatException` 初始化参数（默认 `true`）。
- **请求转储**：`dumpRequest(request)` 返回 `方法 + URI + 查询串`，便于日志。

自定义 Filter 时继承它并实现 `doFilter(HttpServletRequest, HttpServletResponse, FilterChain)` 即可。

## 三、giants-web-springmvc 模块

适用于 Spring MVC 4.2+ 与 5.x（本版本依赖 Spring `5.2.25.RELEASE`）。核心是统一响应封装、统一异常解析、
FastJson 转换器与一组参数解析器。

> 前置条件：统一响应/异常提示信息依赖 Spring 的 `ResourceBundleMessageSource`。请确保容器中存在该 Bean，
> 且资源文件里配置了 `operation.success`、`operation.systemerror` 等消息 key。

### 3.1 统一响应结构 JsonResult

`com.giants.web.springmvc.json.JsonResult` 是统一响应对象，内置 Swagger 注解：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `code` | `byte` | 错误编码，默认 0（成功）。取值见[第五章](#五异常体系与错误编码) |
| `message` | `String` | 提示信息 |
| `businessErrorKey` | `String` | 业务异常 KEY，出现业务异常时返回，前端可基于此做分支 |
| `errMsg` | `Map<String,String>` | 字段校验错误信息，仅校验失败时返回 |
| `data` | `Object` | 业务返回值 |

### 3.2 JsonResultResponseAdvice：统一封装响应

`com.giants.web.springmvc.advice.JsonResultResponseAdvice` 实现 `ResponseBodyAdvice`，
自动把 Controller 的返回值封装进 `JsonResult`。HTTP 状态 200 时 `code=0`、`message=operation.success`；
否则 `code=127`、`message=operation.systemerror`。同时内置 JSONP 支持。

**配置参数**

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| `uriExcludeList` | `List<String>` | 排除的 URI 列表，命中的请求不做统一封装，直接返回原始 body |
| `jsonpQueryParamNames` / `jsonpQueryParamName` | `String[]` / `String` | JSONP 回调参数名。命中且值合法（`[0-9A-Za-z_.]*`）时以 `application/javascript` 输出 |

**配置示例（XML）**

```xml
<bean class="com.giants.web.springmvc.advice.JsonResultResponseAdvice">
    <property name="jsonpQueryParamNames">
        <array>
            <value>callback</value>
        </array>
    </property>
    <property name="uriExcludeList">
        <list>
            <value>/api/file/download</value>
        </list>
    </property>
</bean>
```

**效果**

```java
@RestController
@RequestMapping("/user")
public class UserController {
    @GetMapping("/{id}")
    public User get(@PathVariable Long id) {
        return userService.load(id); // 直接返回业务对象
    }
}
```

实际响应会被自动封装为：

```json
{ "code": 0, "message": "操作成功!", "data": { "id": 1, "name": "..." } }
```

> `JsonpResultResponseAdvice` 为已废弃（`@Deprecated`）的旧类，JSONP 能力已并入 `JsonResultResponseAdvice`，新项目无需使用。

### 3.3 FastJsonHttpMessageConverter：JSON 转换器

`com.giants.web.springmvc.json.FastJsonHttpMessageConverter` 适用于 Spring MVC 4.2 以上版本。
当 FastJson 版本较低（如 1.2.22）时，官方转换器可能存在兼容性问题，可用此转换器替代。它还内置了对 `JsonpResult` 的序列化支持（输出 `callback(...)` 形式）。

**配置参数**

| 参数 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `charset` | `java.nio.charset.Charset` | UTF-8 | 字符集 |
| `features` | `SerializerFeature[]` | `new SerializerFeature[0]` | FastJson 序列化特性 |
| `filters` | `SerializeFilter[]` | `new SerializeFilter[0]` | 序列化过滤器组 |
| `jsonProcessInterceptors` | `JsonProcessInterceptor[]` | — | JSON 读写前后拦截器 |
| `dateFormat` | `String` | — | 日期时间格式化 |

**配置示例**

```xml
<mvc:annotation-driven>
    <mvc:message-converters register-defaults="true">
        <bean class="com.giants.web.springmvc.json.FastJsonHttpMessageConverter">
            <property name="dateFormat" value="yyyy-MM-dd HH:mm:ss"/>
            <property name="features">
                <array>
                    <value>WriteMapNullValue</value>
                    <value>WriteNullStringAsEmpty</value>
                </array>
            </property>
            <property name="filters">
                <array>
                    <bean class="com.giants.web.springmvc.json.JsonSerializePropertyFilter">
                        <property name="ignorePropertyNames">
                            <list>
                                <value>password</value>
                            </list>
                        </property>
                    </bean>
                </array>
            </property>
        </bean>
    </mvc:message-converters>
</mvc:annotation-driven>
```

### 3.4 JsonProcessInterceptor：JSON 读写拦截器

`com.giants.web.springmvc.json.JsonProcessInterceptor` 接口提供 JSON 读写前后的钩子，
配合 `FastJsonHttpMessageConverter` 使用，可用于日志、加解密、埋点等：

- `readBefore(byte[] jsonBytes)` / `readAfter(Object object)`：反序列化前后。
- `writeBefore(Object object)` / `writeAfter(String jsonStr)`：序列化前后。

### 3.5 JsonSerializePropertyFilter：属性过滤

`com.giants.web.springmvc.json.JsonSerializePropertyFilter` 实现 FastJson 的 `PropertyFilter`，
通过 `ignorePropertyNames` 指定需要在序列化时忽略的属性名（如敏感字段 `password`）。

### 3.6 异常解析器

模块提供两个 `HandlerExceptionResolver`，负责把 Controller 抛出的异常翻译成统一 JSON 输出。
两者都通过 `BuildExceptionJsonResult` 把 `GiantsException` 体系映射为对应错误码与国际化提示（详见[第五章](#五异常体系与错误编码)）。

#### JsonResultExceptionResolver（推荐）

`com.giants.web.springmvc.resolver.JsonResultExceptionResolver`：把异常解析为 `JsonResult` 对象并序列化输出。
仅对返回 JSON 的处理方法生效（`@RestController` 或标注 `@ResponseBody` 的方法；`includeModelAndView=true` 时对所有方法生效）。

| 参数 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `messageConverters` | `List<HttpMessageConverter<Object>>` | — | HTTP 输出信息转换器 |
| `includeModelAndView` | `boolean` | `false` | 是否对返回 `ModelAndView` 的方法也进行解析 |
| `jsonpQueryParamNames` / `jsonpQueryParamName` | `String[]` / `String` | — | JSONP 回调参数名 |

#### JsonExceptionResolver

`com.giants.web.springmvc.resolver.JsonExceptionResolver`：功能类似，额外提供 `responseExceptionStatus`
（HTTP 响应状态码，默认 600）用于标识异常响应。

| 参数 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `messageConverters` | `List<HttpMessageConverter<Object>>` | — | HTTP 输出信息转换器 |
| `includeModelAndView` | `boolean` | `false` | 是否对返回 `ModelAndView` 的方法进行解析 |
| `responseExceptionStatus` | `int` | `600` | HTTP 响应状态码 |
| `jsonpQueryParamNames` | `String[]` | — | JSONP 回调参数名 |

**配置示例**

```xml
<bean class="com.giants.web.springmvc.resolver.JsonResultExceptionResolver">
    <property name="messageConverters">
        <list>
            <bean class="com.giants.web.springmvc.json.FastJsonHttpMessageConverter"/>
        </list>
    </property>
</bean>
```

### 3.7 ControllerValidationAop：输入数据校验

`com.giants.web.springmvc.aop.ControllerValidationAop` 是数据校验切面，配合 `spring-modules-validation`
（`DefaultBeanValidator`）使用。校验不通过时抛出 `com.giants.common.exception.DataValidationException`，
再由异常解析器统一输出字段错误信息。

> 前提：被切入的 Controller 方法参数约定为 `(model, BindingResult)`，切面取 `args[0]` 为待校验对象、`args[1]` 为 `BindingResult`。

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| `validator` | `org.springmodules.validation.commons.DefaultBeanValidator` | 数据验证器 |
| `errorMessageKey` | `String` | 校验失败提示信息的资源 key |
| `dontThrowExceptionsReturnTypes` | `List<String>` | 不抛异常的返回值类型列表（如 `org.springframework.web.servlet.ModelAndView`），命中则只校验不抛异常 |

**配置示例**

```xml
<bean id="controllerValidationAop" class="com.giants.web.springmvc.aop.ControllerValidationAop">
    <property name="validator" ref="beanValidator"/>
    <property name="errorMessageKey" value="request.data.validation.fail"/>
    <property name="dontThrowExceptionsReturnTypes">
        <list>
            <value>org.springframework.web.servlet.ModelAndView</value>
        </list>
    </property>
</bean>

<aop:config>
    <aop:aspect ref="controllerValidationAop">
        <aop:before method="validate"
                    pointcut="execution(* com.example.controller..*(..,org.springframework.validation.BindingResult))"/>
    </aop:aspect>
</aop:config>
```

### 3.8 参数解析器

一组 `HandlerMethodArgumentResolver`，把 Cookie / Session 的值直接注入 Controller 方法参数，实现业务代码与 Servlet API 解耦。
**匹配规则是「参数类型 + 参数名同时相等」**，因此需要开启参数名保留（编译加 `-parameters`）或确保能获取参数名。

#### CookieHandlerMethodArgumentResolver

按 `cookieName` 注入对应 Cookie 值到 `String` 类型且同名的参数。

```xml
<mvc:annotation-driven>
    <mvc:argument-resolvers>
        <bean class="com.giants.web.springmvc.resolver.CookieHandlerMethodArgumentResolver">
            <property name="cookieName" value="token"/>
        </bean>
    </mvc:argument-resolvers>
</mvc:annotation-driven>
```

```java
@GetMapping("/profile")
public User profile(String token) { ... } // token 自动来自 Cookie
```

#### SessionHandlerMethodArgumentResolver（Spring MVC 4.x）

按 `SessionAttribute`（`name` + `type`）从 Session 注入参数。

```xml
<bean class="com.giants.web.springmvc.resolver.SessionHandlerMethodArgumentResolver">
    <property name="sessionAttribute">
        <bean class="com.giants.web.springmvc.resolver.SessionAttribute">
            <property name="name" value="loginUser"/>
            <property name="type" value="com.example.model.User"/>
        </bean>
    </property>
</bean>
```

```java
@GetMapping("/home")
public String home(User loginUser) { ... } // 从 Session 注入
```

#### SessionArgumentResolver（Spring MVC 3.x）

适用于 Spring MVC 3.x 的 `WebArgumentResolver`，支持配置多个 `SessionAttribute`（`sessionAttributes` / `sessionAttribute`）。

## 四、giants-web-struts2 模块

适用于 Struts2 2.5.x（本版本依赖 `2.5.33`）。核心是 JSON Action 基类、CRUD Action 骨架，以及配套的校验/异常拦截器。

### 4.1 JsonActionSupport：JSON 请求基类

`com.giants.web.struts2.JsonActionSupport` 封装统一返回字段，供 JSON 结果视图（如 `struts2-json-plugin`）序列化：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `success` | `boolean`（默认 `true`） | 是否处理成功 |
| `errorCode` | `byte`（默认 0） | 错误编码 |
| `message` | `String` | 提示信息 |
| `fieldValidationError` | `Map<String,List<String>>` | 字段校验错误 |
| `validators` | `List<GiantsValidator>` | `validator()` 输出的校验规则 |

**`validator()` 方法**：动态读取当前 Action 配置的所有 Struts2 校验规则，转换为 `GiantsValidator` 列表返回（结果码 `validator`），
可让前端复用后端校验规则做客户端校验。若无任何校验器，`success=false`、`message` 取 `not.find.validator`。

### 4.2 JsonAction：进一步简化

`com.giants.web.struts2.JsonAction` 继承 `JsonActionSupport`，业务逻辑只需实现 `executeAction()`：

```java
public class SaveUserAction extends JsonAction {
    private User user;

    @Override
    public void executeAction() throws Exception {
        userService.save(user); // 抛出的异常会被统一处理
    }

    public void setUser(User user) { this.user = user; }
}
```

- 执行成功：`success=true`、`errorCode=0`，返回 `success`。
- 抛出异常：交由 `BusinessExceptionHandler.handleExceptionToJson` 处理（设置 `success=false` 与对应错误码/消息），返回 `error`。

### 4.3 CRUDAction\<T\>：增删改查骨架

`com.giants.web.struts2.CRUDAction<T>` 实现 `Preparable`，内置分页对象 `com.giants.common.tools.Page<T>`（默认第 1 页、每页 5 条）
和查询过滤条件 `queryFilters`。子类实现数据访问抽象方法即可获得标准 CRUD 动作。

**需实现的抽象方法**

| 方法 | 说明 |
| --- | --- |
| `int listCount(Map<String,Object> queryFilters)` | 查询总记录数 |
| `List<T> listResultSet(Map<String,Object> queryFilters)` | 查询结果集 |
| `void createEntity()` | 新增实体 |
| `void modifyEntity()` | 修改实体 |
| `void loadEntity(Serializable id)` | 加载实体 |
| `void deleteEntity()` | 删除实体 |

**内置动作方法与结果码**

| 方法 | 结果码 | 说明 |
| --- | --- | --- |
| `list()` / `execute()` | `success` | 分页查询；自动把 `offset`、`size` 放入 `queryFilters` |
| `input()` | `input` | 进入表单页 |
| `save()` | `reload` / `input` | `id` 为空则新增否则修改；业务异常返回 `input` |
| `delete()` | `reload` | 删除 |

`prepareInput()` / `prepareSave()` / `prepareDelete()` 会在对应动作前调用 `prepareModel()`，当 `id` 非空时预加载实体。

**示例**

```java
public class UserAction extends CRUDAction<User> {
    private User model;

    @Override protected int listCount(Map<String,Object> f) { return userDao.count(f); }
    @Override protected List<User> listResultSet(Map<String,Object> f) { return userDao.query(f); }
    @Override protected void createEntity() { userDao.insert(model); }
    @Override protected void modifyEntity() { userDao.update(model); }
    @Override protected void loadEntity(Serializable id) { this.model = userDao.load(id); }
    @Override protected void deleteEntity() { userDao.delete(getId()); }
}
```

### 4.4 拦截器

#### JsonActionInterceptor：异常拦截

`com.giants.web.struts2.interceptor.JsonActionInterceptor` 拦截 `JsonActionSupport` 及子类的调用，
捕获执行期异常并交由 `BusinessExceptionHandler.handleExceptionToJson` 统一处理，返回 `error`。

#### JsonValidationInterceptor：校验拦截

`com.giants.web.struts2.interceptor.JsonValidationInterceptor` 继承 `AnnotationValidationInterceptor`。
当校验产生字段错误时，自动设置 `success=false`、`errorCode=2`（数据验证失败）、`message` 取 `request.data.validation.fail`，
并把字段错误写入 `fieldValidationError`。

**拦截器栈配置示例（struts.xml）**

```xml
<interceptors>
    <interceptor name="jsonAction"
        class="com.giants.web.struts2.interceptor.JsonActionInterceptor"/>
    <interceptor name="jsonValidation"
        class="com.giants.web.struts2.interceptor.JsonValidationInterceptor"/>

    <interceptor-stack name="jsonStack">
        <interceptor-ref name="jsonAction"/>
        <interceptor-ref name="defaultStack">
            <param name="validation.excludeMethods">validator</param>
        </interceptor-ref>
        <interceptor-ref name="jsonValidation"/>
    </interceptor-stack>
</interceptors>

<default-interceptor-ref name="jsonStack"/>
```

### 4.5 WebUtils（Struts2）

`com.giants.web.struts2.WebUtils` 基于 `ActionContext` / `ServletActionContext` 提供 Session 与请求访问：

| 方法 | 说明 |
| --- | --- |
| `getSession()` | 获取 Session Map |
| `getSessionAttribute(name)` / `setSessionAttribute(name, value)` | 读写 Session 属性 |
| `getRequest()` | 获取当前 `HttpServletRequest`（无上下文时返回 `null`） |
| `getRemoteAddr()` | 获取客户端真实 IP（依次尝试 `X-Real-IP`、`X-Forwarded-For`、各类代理头，最后回退 `getRemoteAddr()`） |

> 注意：这是 Struts2 专用的 `WebUtils`（`com.giants.web.struts2.WebUtils`），与 `giants-web` 模块的 `com.giants.web.utils.WebUtils` 是两个不同的类。

## 五、异常体系与错误编码

框架的异常处理建立在 `giants-common` 的 `GiantsException` 抽象类之上。业务代码抛出语义化异常，
框架（`BuildExceptionJsonResult` / `BuildJsonErrorMsg`）负责翻译为错误码与提示信息。

### 5.1 异常与错误码对应关系

| 异常类 | code | 常量 | 含义 |
| --- | --- | --- | --- |
| — | 0 | `ERROR_CODE_SUCCESS` | 操作成功 |
| `NotLoggedInException` | 1 | `ERROR_CODE_NOT_LOGGED_IN` | 没有登录，或没有进行认证 |
| `DataValidationException` | 2 | `ERROR_CODE_DATA_CHECK_FAILURE` | 数据验证失败 |
| `BusinessException` | 3 | `ERROR_CODE_BUSINESS_EXCEPTION` | 业务操作异常 |
| `NotAuthorityException` | 4 | `ERROR_CODE_NOT_AUTHORITY` | 没有操作权限 |
| `DataOperationException` | 5 | `ERROR_CODE_DATA_OPERATION_EXCEPTION` | 数据操作异常 |
| `ViewException` | 6 | `ERROR_CODE_VIEW_EXCEPTION` | 显示层异常 |
| `TokenCodeIllegalException` | 7 | `ERROR_CODE_TOKEN_EXCEPTION` | 授权码非法 |
| `SystemException` / 其他未知异常 | 127 | `ERROR_CODE_SYSTEM_ERROR` | 系统错误 |

### 5.2 异常翻译规则

`BuildExceptionJsonResult.build(...)`（Spring MVC）与 `BuildJsonErrorMsg.build(...)`（Struts2）遵循相同逻辑：

1. `UndeclaredThrowableException`：解包后递归处理真实异常。
2. `GiantsException`：取其 `buildErrorCode()` 作为 `code`；`message` 优先用 `errorMessage`，否则按 `errorMessageKey` 从资源文件取国际化文案。
   - `DataValidationException`：把字段错误 `FieldError` 列表翻译为 `errMsg`（字段名 → 提示信息）。
   - `BusinessException`：把 `errorKey` 写入 `businessErrorKey`，供前端做业务分支。
3. 其他异常：记录错误日志，`code=127`，`message` 取 `operation.systemerror`。

### 5.3 业务代码抛异常示例

```java
// 业务异常：带业务 KEY，前端可基于 businessErrorKey 分支处理
throw new BusinessException("USER_NOT_FOUND", "user.not.found", userId);

// 数据校验异常
DataValidationException ex = new DataValidationException("request.data.validation.fail");
ex.addFieldError(ex.createFieldError("email", "email.invalid", null));
throw ex;
```

对应的统一响应（Spring MVC）：

```json
{
  "code": 3,
  "message": "用户不存在",
  "businessErrorKey": "USER_NOT_FOUND",
  "data": null
}
```

### 5.4 需要配置的国际化消息 key

请在资源文件（`ResourceBundleMessageSource`）中配置以下常用 key：

| key | 用途 |
| --- | --- |
| `operation.success` | 统一封装成功时的默认提示 |
| `operation.systemerror` | 系统错误时的默认提示 |
| `request.data.validation.fail` | 数据校验失败提示（Struts2 校验拦截器 / Spring 校验切面） |
| `not.find.validator` | Struts2 未找到校验器时的提示 |

> 若某个 key 未配置，`getMessage` 会直接返回 key 本身作为兜底，不会抛出异常。

## 六、常见问题（FAQ）

**Q：`WebUtils.getRequest()` 返回 null？**
A：`giants-web` 的 `WebUtils` 依赖 `WebFilter` 把 request 存入 ThreadLocal。请确认 `WebFilter` 已在 `web.xml` 注册且 `url-pattern` 覆盖了当前请求；异步线程/线程池中无法读取到主线程的 ThreadLocal。

**Q：参数解析器（Cookie/Session）不生效？**
A：解析器要求「参数类型 + 参数名同时匹配」。Java 8 默认不保留方法参数名，请在编译时加 `-parameters`（或确保 Spring 能通过其他方式获取参数名），并确认参数名与配置的 `cookieName` / `SessionAttribute.name` 完全一致。

**Q：统一封装后想让个别接口返回原始数据怎么办？**
A：把该接口 URI 加入 `JsonResultResponseAdvice` 的 `uriExcludeList`。

**Q：FastJson 版本较低出现兼容问题？**
A：使用本模块的 `FastJsonHttpMessageConverter` 替代官方转换器（适用于 Spring MVC 4.2+）。

**Q：统一响应/异常提示的文案从哪来？**
A：来自容器中的 `ResourceBundleMessageSource`。请确保该 Bean 存在，并配置了 [5.4 节](#54-需要配置的国际化消息-key) 列出的消息 key。
